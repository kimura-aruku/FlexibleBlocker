# FlexibleBlocker 設計仕様

## 1. 設計方針

FlexibleBlocker は、ビルドツールや外部ライブラリを使用しないChrome Extension Manifest V3の拡張機能として構成する。

利用者の設定はChrome内に保存し、ブロック判定も利用者のブラウザ内で完結させる。

## 2. システム構成

| コンポーネント | ファイル | 責務 |
|---|---|---|
| マニフェスト | `manifest.json` | 権限、Service Worker、画面、Content Scriptの登録 |
| Service Worker | `background.js` | ページ遷移の監視、URLと時間帯の判定、ブロック画面への遷移 |
| 共通処理 | `common.js` | URL解析、URL一致判定、時間帯判定、設定登録の共通処理 |
| ポップアップ | `popup.html`、`popup.js` | 表示中サイトのブロック設定、設定画面への導線 |
| 設定画面 | `options.html`、`options.js` | ブロック設定の追加、一覧表示、時間変更、削除 |
| ブロック画面 | `block.html`、`block.js` | ブロック情報の表示、設定画面への導線、設定解除 |
| Content Script | `content.js` | 対象ページでのContent Script読み込み |
| スタイル | `styles.css` | 各画面で使用する共通表示 |

## 3. 処理フロー

### 3.1 設定登録

1. ポップアップまたは設定画面でURLと時間帯を入力する。
2. URLからプロトコルと先頭の `www.` を除去する。
3. 必要に応じてURLのパス階層を選択する。
4. 同一URLが登録済みでないことを確認する。
5. `chrome.storage.local` の `blockedSites` に設定を保存する。

### 3.2 ブロック判定

1. Service Workerが `webNavigation` またはタブ更新イベントを受け取る。
2. メインフレーム以外と拡張機能内部のURLを除外する。
3. `blockedSites` をChromeローカルストレージから取得する。
4. アクセス先のホスト名とパスを登録済みルールと比較する。
5. URLが一致した場合、現在時刻が設定時間帯に含まれるか判定する。
6. 条件が一致したタブを `block.html` へ遷移させる。

### 3.3 ブロック解除

1. 設定画面またはブロック画面から解除操作を受け取る。
2. 対象ルールを `blockedSites` から削除する。
3. 更新後の配列をChromeローカルストレージへ保存する。

## 4. データ設計

永続データには `chrome.storage.local` を使用する。

```text
blockedSites: Array<{
  url: string,
  fromTime: string,
  toTime: string
}>
```

- `url` はプロトコルを含めない。
- `fromTime` と `toTime` は `HH:MM` 形式とする。
- データ移行を伴うスキーマ変更時は、旧形式との互換性を考慮する。

## 5. URL判定設計

- ホスト名は先頭の `www.` を除去した後、完全一致で比較する。
- パスは `/` ごとの階層に分割して前方一致で比較する。
- クエリ文字列とフラグメントは判定対象に含めない。
- 不正なURLはブロック対象外として扱い、処理を継続する。

## 6. 時間帯判定設計

時刻は分単位に変換して比較する。

- 開始時刻が終了時刻以前: `開始 <= 現在 <= 終了`
- 開始時刻が終了時刻より後: `現在 >= 開始` または `現在 <= 終了`
- `00:00` から `23:59`: 終日

## 7. セキュリティとプライバシー

- ブロック設定と判定処理は利用者のChrome内で完結させる。
- 外部サーバーへの設定データ送信機能は設けない。
- `innerHTML` を利用する箇所では、利用者入力をHTMLとして直接展開しない。
- 閲覧URLやブロック設定などのデバッグログは、公開版に不要な場合は削除または無効化する。
- `<all_urls>` は任意サイトを判定するために必要だが、READMEで用途を明示する。

## 8. 開発・配布

- ビルド工程は設けず、リポジトリのファイルをそのままChromeへ読み込める構成とする。
- 開発時は `chrome://extensions` のデベロッパーモードから読み込む。
- 公開版はChromeウェブストアで配布する。
- バージョンは `manifest.json` の `version` で管理する。

## 9. 技術的な留意事項

- URL判定と時間帯判定は `background.js` と `common.js` の双方に存在するため、仕様変更時は実装差異が生じないよう注意する。
- Service Workerは停止・再起動される前提とし、永続化が必要な情報をメモリだけに保持しない。
- Chrome管理画面など、拡張機能から制御できないURLは処理対象外とする。
