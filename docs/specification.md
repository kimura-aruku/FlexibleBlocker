# FlexibleBlocker 詳細仕様

## 1. 概要

FlexibleBlocker は、ユーザーが指定したWebサイトへのアクセスを、設定した時間帯に制限するChrome拡張機能である。

ブロック対象のURLへ制限時間中にアクセスした場合、対象ページの代わりに拡張機能内の警告ページを表示する。

## 2. 対象環境

- Google Chrome
- Chrome Extension Manifest V3
- 日本語UI

## 3. 機能仕様

### 3.1 ブロック対象の登録

ユーザーは、次の画面からブロック対象を登録できる。

- 拡張機能のポップアップ画面
- 拡張機能の設定画面

登録時には、対象URLとブロックする開始時刻・終了時刻を指定する。

URLに複数のパス階層が含まれる場合、画面上でURLの階層を選択し、ブロック対象とする範囲を指定できる。

### 3.2 URLの判定

URLの判定では、`http://` または `https://` と、ホスト名先頭の `www.` を除外して比較する。

次の単位でブロック対象を指定できる。

1. ドメイン全体
   - 例: `example.com`
2. 指定したパス階層以下
   - 例: `example.com/video` は、`example.com/video` およびその下位パスに一致する
3. より深い特定パス
   - 例: `example.com/video/list`

ホスト名は完全一致で判定し、異なるサブドメインは同一サイトとして扱わない。

### 3.3 時間帯の判定

- 時刻は `HH:MM` 形式で指定する。
- `00:00` から `23:59` を指定した場合は終日ブロックする。
- 開始時刻が終了時刻以前の場合は、同日内の時間帯として判定する。
- 開始時刻が終了時刻より後の場合は、日をまたぐ時間帯として判定する。
  - 例: `22:00` から `06:00`
- 開始時刻と終了時刻はブロック対象時間に含む。
- 判定には利用者端末の現在時刻を使用する。

### 3.4 ブロック時の動作

現在のURLと時刻が登録済みルールに一致した場合、対象タブを拡張機能内のブロック画面へ遷移させる。

ブロック画面には次の情報と操作を表示する。

- サイトがブロック中であること
- ブロック対象URL
- ブロック対象時間帯
- 設定画面を開く操作
- 該当するブロック設定を解除する操作

### 3.5 登録済み設定の管理

設定画面では次の操作ができる。

- 登録済みブロック対象の一覧表示
- 登録件数の表示
- ブロック時間帯の変更
- ブロック設定の削除

同一URLのルールは重複登録しない。

## 4. データ仕様

ブロック設定は `chrome.storage.local` の `blockedSites` に配列として保存する。

各設定は次の値を持つ。

| 項目 | 型 | 内容 |
|---|---|---|
| `url` | 文字列 | プロトコルと先頭の `www.` を除いたブロック対象 |
| `fromTime` | 文字列 | ブロック開始時刻（`HH:MM`） |
| `toTime` | 文字列 | ブロック終了時刻（`HH:MM`） |

設定データは利用者のChrome内に保存し、外部サーバーへ送信しない。

## 5. 権限

| 権限 | 用途 |
|---|---|
| `storage` | ブロック設定の保存と読み込み |
| `webNavigation` | ページ遷移の検知 |
| `activeTab` | 操作中タブのURL取得と制御 |
| `tabs` | 対象タブの取得とブロック画面への遷移 |
| `<all_urls>` | 任意のWebサイトをブロック対象として判定するため |

## 6. 導入方法

- Chromeウェブストアから導入できる。
- GitHubからソースコードを取得し、デベロッパーモードでパッケージ化されていない拡張機能として導入できる。

具体的な導入手順は `README.md` に記載する。

## 7. 対象外・将来の拡張候補

次の機能は現行仕様には含めず、将来の拡張候補とする。

- 曜日ごとのブロック時間設定
- 利用回数や利用時間による制限
- パスワードによる設定変更の保護
- 一時的なブロック解除
